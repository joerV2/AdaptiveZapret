# AdaptiveZapret

Requirements:
* Windows 7 x64, Windows server 2008 R2 (ESU updates or windivert files replacement required)
* Windows 8+ x64, Windows server 2012+ (may require to disable secure boot)
* Windows 11+ ARM64, Windows server 2025+ (requires testsigning mode)
* Windows Server requires installing of wireless networking feature

### Что это

Данный проект - некоторое переосмысление работы с zapret. По крайней мере для меня.

В основу легли проекты [zapret-win-bundle](https://github.com/bol-van/zapret-win-bundle) и [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube), первый как крепкий фундамент, на основе которого вся эта хрень и работает, но сложный для понимания обычному пользователю, а второй - как более "однокнопочное" решение, но тряющее из-за этого гибкость.

### 🚀 Быстрый старт

1. Скачайте последний релиз из раздела [Releases](https://github.com/joerV2/AdaptiveZapret/releases).
2. Распакуйте архив в любую папку (путь не должен содержать кириллицу).
3. Запустите `AdaptiveZapret.exe` от имени администратора.
4. В главном окне нажмите **«Включить сниффер»** и попробуйте подключиться к сайтам, которые нужно обходить или добавьте их вручную.
5. После появления доменов в таблице и окончания подбора стратегий, нажмите **«Обновить списки»** и **«Запустить zapret»**.
6. Если что-то не открывается, попробуйте выбрать другую найденную стратегию.
7. Готово. Если нужно остановить – нажмите **«Остановить zapret»**. Не забывайте останавливать zapret при выполнении повторных поисков стратегий, иначе сайт будет отмечен как "чистый".

### Краткое описание работы

Главная задумка AdaptiveZapret - его адаптация к условиям работы. В отличие от использования готовых стратегий [zapret-discord-youtube](https://github.com/Flowseal/zapret-discord-youtube), данный проект нацелен на самостоятельный поиск рабочих вариантов. Для полноценной работы AdaptiveZapret необходимо запускать без активного zapret, без активных VPN или любых других способов обхода, иначе от него банально не будет результата.

При первом запуске база ссылок и стратегий для них будет пуста, необходимо включить сниффер, который будет автоматически закидывать в базу все домены, к которым ваш ПК пытается подключаться и проводить для них проверку на доступность и поиск стратегий обхода.

Для корректной работы с браузерами необходимо отключать DoH и встроенные в браузеры DNS, кнопки для этого есть. К сожалению, без этого никак, ибо в отличие от других программ, браузеры всё же шифруют трафик, из-за чего перехват его для получения ссылок становится невозможным. Если нет желания или необходимости в работе с абсолютно всеми ссылками из браузера, можно добавить нужные в таблицу вручную.

Все ссылки, попавшие в базу, начнут автоматически проверяться, процесс не очень быстрый, может уходить от 30 секунд до 10 минут на проверку одного домена. Проверка выполняется в quic режиме, поэтому будут использоваться первые найденные стратегии, для большинства случаев этого достаточно. В таблице доменов есть возможность выбора, с какой стратегией он будет открываться. Если какой-то ресурс плохо работает с найденными стратегиями, проведите повторное сканирование ~~или переключите режим на standard, это более глубокое сканирование, но требует гораздо больше времени ~20-30 минут на один домен~~(реализую переключение чуть позже).

### АНТИВИРУСЫ
windivert может вызвать реакцию антивируса.
windivert - это инструмент для перехвата и фильтрации трафика, необходимый для работы zapret.
Замена iptables и NFQUEUE в Linux, которых нет под Windows.
Он может использоваться как хорошими, так и плохими программами, но сам по себе не является вирусом.
Драйвер windivert64.sys подписан для возможности загрузки в 64-битное ядро windows.
Любой желающий может сравнить файлы с [оригиналами](https://reqrypt.org/download) от автора.
Но антивирусы склонны относить подобное к классам повышенного риска или хакерским инструментам.
В случае проблем используйте исключения или выключайте антивирус совсем.
